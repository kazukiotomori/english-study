# 英語学習効率化アプリ 仕様書（Draft）

## 1. コンセプト

「一つの例文を多角的にしゃぶり尽くす」ことで、記憶の定着と「使える英語」への変換を最短距離で実現する、高効率サイクル型学習アプリ。

## 2. システム構成・技術要件

* **ターゲット:** スマートフォン（縦長レスポンシブデザイン）。
* **データ管理:** * テキストデータ（英文・和訳・単語）はすべて**JSONファイル**で保持。
* 音声データは**ローカルの音声ファイル**（MP3等）を読み込み。


* **AI非搭載:** 外部API通信を介さず、レスポンスの速さとオフライン動作を重視。
* **ナビゲーション:** 章（Chapter 8, 9...）およびセクション（Reading Aloud 8-1...）を選択できる階層構造。

## 3. 学習フロー（4つのステップ）

各コンテンツ（例文）に対し、以下の順番で学習を進行させる。

### STEP 1：単語熟語（Input）

* **形式:** 4択クイズ。
* **挙動:** * 正解タップで「○」表示後、自動で次へ（爆速UI）。
* 不正解時は「×」と正解をハイライト表示。


* **モード:** 「英→日」をデフォルトとし、設定で「日→英」への切り替えが可能。
* **ロジック:** ダミー選択肢は同じ章内の単語からランダム抽出。

### STEP 2：音読（Practice）

* **位置付け:** アプリの最重要ステップ。
* **表示:** 英文を大きく中央に配置。
* **機能:** * 音声再生ボタン、再生速度変更（1.0x, 0.8x等）。
* とにかく反復して声に出すことを促すシンプルなUI。



### STEP 3：英文和訳（Decoding）

* **形式:** 瞬間和訳。
* **挙動:** 英文を見て和訳を頭で浮かべ、「答えを表示」ボタンで確認。

### STEP 4：和文英訳（Encoding）

* **形式:** 瞬間英訳。
* **挙動:** 和文を見て英文を構築し、音声再生またはテキスト表示で答え合わせ。
* **モード:** STEP 3と4はトグルスイッチで「デフォルト：英文和訳」から「逆モード：和文英訳」へ切り替え可能。

## 4. データ構造（JSONサンプル）

画像 の「READING ALOUD 8-1」および「8-3」に基づいた構造例：

```json
{
  "chapter": 8,
  "sections": [
    {
      "id": "8-1",
      "title": "Mr. Oguchi's New Year's Eve",
      "sentence_en": "At the end of last year, Mr. Oguchi went to his grandmother's house. ... (以下、画像の内容)",
      "sentence_ja": "去年の終わりに、小口さんは祖母の家に行きました。...",
      "audio_file": "audio_8_1.mp3",
      "vocabulary": [
        { "en": "at the end of", "ja": "〜の終わりに" },
        { "en": "tiny village", "ja": "とても小さな村" },
        { "en": "housework", "ja": "家事" }
      ]
    },
    {
      "id": "8-3",
      "title": "The Hare and the Tortoise",
      "sentence_en": "One day, a hare and a tortoise were walking in the forest.",
      "sentence_ja": "ある日、ウサギとカメが森を歩いていました。",
      "audio_file": "audio_8_3.mp3",
      "vocabulary": [
        { "en": "hare", "ja": "野うさぎ" },
        { "en": "tortoise", "ja": "かめ" },
        { "en": "steadily", "ja": "着実に、どんどん" }
      ]
    }
  ]
}

```

## 5. UI/UXのこだわり

* **片手操作:** すべてのアクションボタン（再生、次へ、答えを表示）を親指が届く下半分に配置。
* **進捗管理:** 章ごとのクリア状況を可視化。
* **設定の保持:** モード切替（英日逆転など）の状態はアプリ終了後も保持。
